# Research Questions

## RQ-001: 图数据库 vs 关系型数据库选型

**问题：** 原子标签 + 自由组合的知识图谱，底层存储用什么？

**候选方案：**
- 图数据库（Neo4j, TigerGraph, ArangoDB）— 天然适合关系遍历
- 属性图 on 关系型 DB（PostgreSQL + JSONB + 递归查询）— 运维简单
- 文档数据库 + 图层（MongoDB + 手动关系管理）
- 向量数据库（语义搜索用，但不适合结构化关系）

**考量维度：**
- 查询模式：多跳关系遍历 vs 标签交叉筛选
- 数据规模：几千到几万实体（钢笔领域有限）
- 运维复杂度：远程 agent 能否方便部署
- AI 集成：embedding 存储、语义搜索能力
- 成本：免费 tier / 自部署成本

## RQ-002: 前端漫游体验技术方案

**问题：** 用什么技术实现"逛维基百科"式的链接漫游 + 局部图可视化？

**候选方案：**
- 静态站点生成（类 Obsidian Publish / Wiki.js）+ 图可视化插件
- 全栈 Web App（Next.js/Nuxt）+ D3.js/Cytoscape.js 图可视化
- Obsidian 本身（本地优先，插件生态丰富，但不方便远程部署）

**考量维度：**
- 漫游体验的流畅度
- 能否嵌入富媒体（图片、视频）
- 局部图可视化的交互性
- 部署与分享方便性
- 远程 agent 开发友好度

## RQ-003: AI 标注 pipeline 设计

**问题：** 如何把非结构化内容（测评文章、书籍章节、论坛帖子）自动拆解为原子标签？

**需要研究：**
- 标签体系设计（taxonomy）：如何定义"最小颗粒度"
- NER + 关系抽取：用 LLM 还是传统 NLP
- 人机协作：AI 标注 + 人工审核的工作流
- 增量入库：新内容进来如何与现有图谱对齐
